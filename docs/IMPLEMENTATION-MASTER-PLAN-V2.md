**RLS Policies**

```sql
-- Enable RLS
ALTER TABLE public.profiles ENABLE ROW LEVEL SECURITY;
ALTER TABLE public.user_journeys ENABLE ROW LEVEL SECURITY;
ALTER TABLE public.opportunities ENABLE ROW LEVEL SECURITY;
ALTER TABLE public.revenue ENABLE ROW LEVEL SECURITY;
ALTER TABLE public.templates ENABLE ROW LEVEL SECURITY;

-- Profiles: Users can only access their own
CREATE POLICY "Users can view own profile"
  ON public.profiles FOR SELECT
  USING (auth.uid() = id);

CREATE POLICY "Users can update own profile"
  ON public.profiles FOR UPDATE
  USING (auth.uid() = id);

-- User Journeys: Users can only access their own
CREATE POLICY "Users can view own journey"
  ON public.user_journeys FOR SELECT
  USING (auth.uid() = user_id);

CREATE POLICY "Users can update own journey"
  ON public.user_journeys FOR UPDATE
  USING (auth.uid() = user_id);

CREATE POLICY "Users can insert own journey"
  ON public.user_journeys FOR INSERT
  WITH CHECK (auth.uid() = user_id);

-- Opportunities: Public read for active
CREATE POLICY "Opportunities are public read"
  ON public.opportunities FOR SELECT
  TO PUBLIC
  USING (is_active = TRUE);

-- Revenue: Users can only access their own
CREATE POLICY "Users can view own revenue"
  ON public.revenue FOR SELECT
  USING (auth.uid() = user_id);

CREATE POLICY "Users can insert own revenue"
  ON public.revenue FOR INSERT
  WITH CHECK (auth.uid() = user_id);

-- Templates: Public read for active
CREATE POLICY "Templates are public read"
  ON public.templates FOR SELECT
  TO PUBLIC
  USING (is_active = TRUE);
```

  clear(): Promise<boolean>;

**CacheManager**
```typescript
export class CacheManager {
  async getOrCompute<T>(
    key: string,
    compute: () => Promise<T>,
    ttlMs?: number
  ): Promise<T>;
  
  async invalidate(pattern: string): Promise<number>;
  async getStats(): Promise<CacheStats>;
}
```

**BaseRepository**
```typescript
export abstract class BaseRepository<T extends Entity> {
  protected abstract tableName: string;
  protected abstract transformFromDB(data: unknown): T;
  protected abstract transformToDB(entity: Partial<T>): Record<string, unknown>;
  
  async findById(id: string): Promise<T | null>;
  async findAll(options?: QueryOptions): Promise<T[]>;
  async create(entity: Omit<T, 'id' | 'createdAt' | 'updatedAt'>): Promise<T>;
  async update(id: string, entity: Partial<T>): Promise<T>;
  async delete(id: string): Promise<boolean>;
}
```

## DEPLOYMENT STRATEGY

### Environments

```
Local Development → Staging → Production
     │                  │          │
  Local DB         Staging DB   Production DB
  (Docker)         (Supabase)   (Supabase)
```

### Deployment Pipeline

1. **Local Development**
   - Feature branches
   - Local Supabase (Docker)
   - Hot reload

2. **Pull Request**
   - CI tests run
   - Linting checks
   - Type checking
   - Preview deploy

3. **Staging**
   - Merge to `staging` branch
   - Auto-deploy to staging
   - QA testing
   - Stakeholder review

4. **Production**
   - Merge to `main` branch
   - Auto-deploy to production
   - Smoke tests
   - Monitoring

### Rollback Procedures

**Database Rollback:**
```bash
# Revert last migration
supabase migration revert

# Or restore from backup
supabase db restore <backup-id>
```

**Code Rollback:**
```bash
# Revert commit
git revert <commit-hash>

# Or reset to previous deploy
git reset --hard <previous-commit>
git push --force
```
